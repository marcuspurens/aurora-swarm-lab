# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please report security issues privately and do not open a public issue with exploit details.

Preferred path:

1. Use GitHub Security Advisories for this repository, if available.
2. If advisories are not enabled, open a minimal issue titled `Security contact requested` without sensitive details, and maintainers will provide a private channel.

## What to Include

- Affected component/file
- Reproduction steps
- Impact/severity estimate
- Suggested mitigation (if known)

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Triage and remediation plan: as soon as feasible based on severity and scope
